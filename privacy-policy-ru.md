# Политика конфиденциальности KLYDE

Дата вступления в силу: 07.03.2026

Данная Политика конфиденциальности описывает, какие данные обрабатывает приложение KLYDE, как и для каких целей они используются.

## 1. О приложении

KLYDE — мессенджер с P2P-архитектурой и сквозным шифрованием контента сообщений.

Контакты для вопросов по данным и конфиденциальности: `klyde.app@gmail.com`.

## 2. Какие данные мы обрабатываем

### 2.1 Данные аккаунта и идентификаторы

- `peerId` (идентификатор пользователя, производный от ключа)
- `UIN` (числовой идентификатор для удобного добавления контактов)
- публичный ключ пользователя

### 2.2 Данные профиля

- никнейм
- аватар (если установлен пользователем)

### 2.3 Технические и сервисные данные

- FCM-токен устройства (для push/wakeup уведомлений)
- служебные сетевые данные, необходимые для соединения и доставки (например, метаданные сигналинга)
- данные о статусе Premium (если используется)

### 2.4 Пользовательский контент

- текстовые сообщения
- медиа/файлы, отправленные пользователем

Содержимое сообщений шифруется на стороне клиента (сквозное шифрование).

## 3. Для чего используются данные

Данные используются для:

- регистрации и маршрутизации пользователей (`peerId`, `UIN`)
- установления P2P-соединений и доставки сообщений
- отправки push/wakeup уведомлений (FCM)
- отображения профиля и контактов
- показа рекламы в free-версии (Yandex Mobile Ads)
- хранения/передачи больших вложений через облачное хранилище (Backblaze B2), если применимо

## 4. Сторонние сервисы

Приложение может использовать следующие сторонние сервисы:

- Firebase Cloud Messaging (Google) — push/wakeup
- Yandex Mobile Ads SDK — реклама
- Backblaze B2 — хранение больших зашифрованных вложений

Каждый из сервисов обрабатывает данные в рамках собственных политик конфиденциальности.

## 5. Разрешения Android

Приложение может запрашивать:

- `INTERNET`, `ACCESS_NETWORK_STATE` — сеть и соединение
- `POST_NOTIFICATIONS` — уведомления
- `CAMERA`, `RECORD_AUDIO` — фото/видео/голосовые
- `ACCESS_FINE_LOCATION` / `ACCESS_COARSE_LOCATION` — отправка геопозиции (по действию пользователя)
- `FOREGROUND_SERVICE`, `WAKE_LOCK`, `RECEIVE_BOOT_COMPLETED` — стабильная работа фонового соединения
- разрешения, связанные с точными alarm, для watchdog-механизма доставки

Разрешения используются только для функциональности приложения.

## 6. Хранение данных и безопасность

- Локальные данные (включая ключи, профиль, историю) хранятся на устройстве пользователя.
- Доступна функция резервной копии идентичности с шифрованием по паролю пользователя.
- Контент сообщений предназначен для передачи в зашифрованном виде.

Важно: метаданные соединения (например, время, факт взаимодействия, сетевые параметры) могут обрабатываться на уровне транспортной инфраструктуры.

## 7. Реклама

В бесплатной версии может показываться реклама через Yandex Mobile Ads.

Мы не продаём содержимое ваших сообщений рекламным сетям.

## 8. Сроки хранения

Срок хранения данных зависит от типа данных и настроек пользователя:

- локальная история хранится на устройстве до удаления пользователем/приложением
- служебные серверные данные хранятся в объёме, необходимом для работы функций (например, регистрация UIN, wakeup-механизмы, premium-статус)

## 9. Права пользователя

Пользователь может:

- удалить приложение и локальные данные
- удалить чаты/контакты внутри приложения
- управлять разрешениями в настройках Android
- обратиться по email для вопросов по обработке данных

## 10. Дети

Приложение не предназначено для использования детьми без согласия законных представителей в юрисдикциях, где это требуется.

## 11. Изменения политики

Мы можем обновлять Политику конфиденциальности. Актуальная версия публикуется по месту размещения документа и вступает в силу с указанной даты.
